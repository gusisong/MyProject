# AI 多蛇分裂游戏原型设计文档

## 1. 项目概述
在原有 AI 贪吃蛇基础上，引入多蛇并行、随机分裂、持久化排名和命名机制。取消血量限制，改为一触即死的硬核模式。

## 2. 核心机制优化
*   **取消血量**: 移除 20 HP 限制。任何碰撞（墙壁、自身、其他蛇）立即导致游戏结束。
*   **分裂机制**: 
    *   每条长度 > 2 的蛇在每一帧有 0.5% 的概率触发分裂。
    *   分裂后，原蛇保留前半部分，后半部分变为一条新蛇。
    *   若原长度为奇数 (L)，分裂为 (L+1)/2 和 (L-1)/2，以此作为分裂的长度损耗惩罚。
*   **多蛇 AI**: 每条蛇独立运行 A* 算法。所有蛇的身体均为全局障碍物。

## 3. UI 与数据增强
*   **命名系统**: 初始蛇从预设名录（Apollo, Blaze, Comet, Dash, Echo）中随机取名。分裂出的蛇继承父名并加后缀（如 "Blaze Jr.", "Blaze III"）。
*   **排名机制**: 记录每轮“总存活时间”和“总吃果数”。页面不刷新时，统计前 5 名记录。
*   **视觉**: 在蛇头上方显示其名字。

## 4. 技术实现
*   **数据结构**: `snakes` 数组存储所有活跃蛇对象。每个对象包含 `body`, `name`, `color` 等。
*   **同步逻辑**: 方案 A（统一时间步长），在单次 `gameLoop` 中遍历 `snakes` 进行决策和移动。
*   **渲染**: Canvas 渲染所有蛇，并在头部绘制 Text 标签。

## 5. 死亡与重置
*   任何一条蛇发生碰撞，所有蛇停止活动，进入结算页面，并更新排名列表。
